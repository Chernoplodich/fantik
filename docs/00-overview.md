# 00 · Обзор проекта «Fantik»

## Цель продукта

Telegram-бот, который даёт комьюнити фанфикшена единое место чтобы:

- **Читать** фанфики прямо в Telegram — удобно, быстро, с сохранением авторского форматирования.
- **Публиковать** свои работы после лёгкой, но надёжной модерации.
- **Открывать для себя новое** через умный поиск с фасетами (фандом, возрастной рейтинг, теги, персонажи, варнинги) и подписки на любимых авторов.
- Администрации — получать статистику, запускать рассылки и отслеживать источники трафика.

Бот позиционируется как «читалка + мини-площадка» для русскоязычного фанфик-комьюнити, но архитектура i18n-ready.

## Роли пользователей

| Роль | Права |
|---|---|
| **Читатель** (default) | Читает, ставит лайки, добавляет в избранное, подписывается на авторов, отправляет жалобы. |
| **Автор** | То же + создаёт/редактирует черновики, отправляет на модерацию, видит статистику своих работ. Автор = читатель, установивший `author_nick`. |
| **Moderator** | То же + очередь модерации, принятие/отклонение работ и глав с причиной, управление жалобами. |
| **Admin** | Полные права: роли пользователей, справочники (фандомы, возрастные рейтинги, причины отказа), трекинговые коды, рассылки, баны, глобальные настройки. |

## Ключевые продуктовые решения

1. **Формат принимается из Telegram как есть.** Автор пишет в Telegram, бот хранит текст + MessageEntities (bold, italic, spoiler, link, code, blockquote, custom emoji). Отдача читателю — без потерь, тем же API.
2. **Главы — обязательны.** Даже короткий фик — это фик с одной главой. Унифицирует модель и UX.
3. **Модерация — первая линия защиты.** Публикация проходит через очередь; отказ с причиной; автор дорабатывает и возвращает на повторную модерацию. Любая правка опубликованного фика возвращает его в `pending`.
4. **Поиск — внешний (Meilisearch).** Подробнее в [`08-search-design.md`](08-search-design.md).
5. **Рассылки — через `copyMessage`.** Админ пересылает/пишет сообщение-шаблон, бот отправляет читателям копию с сохранением всего. Подробнее в [`07-broadcast-system.md`](07-broadcast-system.md).
6. **UTM — через `?start=<code>`.** Deep-link создаётся админом, события по воронке записываются и агрегируются. Подробнее в [`10-tracking-analytics.md`](10-tracking-analytics.md).

## Зафиксированные допущения (дефолты)

Эти параметры вшиты в MVP; при необходимости меняются до реализации.

| Параметр | Дефолт |
|---|---|
| Максимум глав на фик | 200 |
| Максимум символов на главу | 100 000 (UTF-16 units) |
| Максимум символов на страницу в читалке | 3 900 (запас от лимита 4 096 под заголовок) |
| Политика правок после публикации | Любая правка → `pending` (без разделения на минорные/мажорные) |
| Таймзона хранения | UTC |
| Таймзона UI по умолчанию | МСК (Europe/Moscow); пользователь может изменить |
| Язык UI | RU (i18n-слой готов, словари — только ru) |
| Обложка | одно фото (JPEG/PNG), max 5 МБ, хранится как `file_id` |
| Длина `author_nick` | 2–32 символа, `a-z0-9_-`, без пробелов |
| Лимит апдейтов от пользователя | 30/мин (middleware-throttle) |
| TTL FSM-состояний | 1 час |
| TTL кэша страниц | 1 час |

## Глоссарий

- **Фик / фанфик** — произведение любительской литературы по вселенной существующего произведения.
- **Фандом** — вселенная/первоисточник (книга, фильм, игра), по которому пишут фики.
- **Возрастной рейтинг** — метка допустимого возраста читателя: G / PG / PG-13 / R / NC-17.
- **Варнинг** — предупреждение о чувствительном содержании (насилие, смерть, ОЖП/ОМП, дарк-контент).
- **Тег** — свободная метка: персонаж, тематика, тип отношений.
- **OC / OMC / OFC** — оригинальный персонаж (мужской/женский).
- **AU** — альтернативная вселенная.
- **Кроссовер** — смешение двух и более фандомов.
- **Драббл / мини / миди / макси** — условные размеры: < 500 слов / < 20 000 / < 70 000 / 70 000+.
- **Entities** — массив `MessageEntity` из Bot API: описывают форматирование без HTML/Markdown.
- **Deep-link** — ссылка `https://t.me/<bot>?start=<payload>` с параметром, который бот получает на `/start`.
