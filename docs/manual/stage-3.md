# Manual smoke-test checklist: Этап 3 · Читалка + пагинатор

Запусти стек локально (`make up && make logs`) и пройди по шагам. Полный Telegram-аккаунт с `ADMIN_TG_IDS` должен быть у тебя на руках, плюс второй аккаунт-автор.

## Подготовка

1. `make migrate` — применяется миграция `0005_reading.py` (таблицы `bookmarks`, `likes`, `reads_completed`, `reading_progress`).
2. Убедись, что `taskiq worker` и `taskiq scheduler` подняты (в compose — контейнеры `worker` и `scheduler`). `LabelScheduleSource` подхватит `outbox_dispatch_tick` с cron `* * * * *` (раз в минуту).
3. `make db-shell` доступен для проверок счётчиков.

## Smoke-тесты

1. **Публикация контентного фика.** Автор создаёт фик с обложкой, одной главой:
   - русский текст + emoji + 👨‍👩‍👧 (ZWJ) + bold + spoiler + code-block + ссылка;
   - 2 абзаца суммарно > 8000 UTF-16 units (чтобы страница точно поделилась).
   Отправляет на модерацию. Админ `/mod_queue` → Approve.
   Ожидание: через ≤ 60 с в `chapter_pages` появляются строки (`SELECT count(*) FROM chapter_pages WHERE chapter_id = X`).

2. **Каталог → чтение.** Другим аккаунтом: `/catalog` → «🆕 Новое» → выбрать фик → появляется `sendPhoto` с обложкой и кнопкой «📖 Читать».

3. **Нажатие «Читать».** Cover удаляется, присылается новое сообщение «Глава 1 · «...» | Страница 1 из Y» + клавиатура (◀ / 📄 / ▶ / ⏮ / 📖 / ⏭ / 📑 / ❤️ / ⚠️).

4. **Листание ◀▶.** Текст обновляется **in-place** через `edit_message_text`. Форматирование (bold, spoiler, ссылки) сохраняется. Emoji/ZWJ не обрезаны посередине.

5. **Длинная глава > 3900 UTF-16.** Навигация корректно показывает `X/Y`, где Y — реальное количество страниц. Последняя страница меньше предыдущих.

6. **Двойной клик ▶.** Быстрые два клика → не должно быть всплывающих ошибок / дубликатов. В логах можно увидеть `message is not modified` — это ожидаемо и подавляется.

7. **Bookmark / Like.**
   - Нажать 📑 → кнопка станет 🔖 (или наоборот). Сообщение-подтверждение сверху.
   - В БД: `SELECT * FROM bookmarks WHERE user_id = <uid>`.
   - Нажать ❤️ → `SELECT likes_count FROM fanfics WHERE id = X` увеличился на 1. Повторный клик — уменьшается (не ниже 0).

8. **Report-заглушка.** Клик ⚠️ → alert «Жалобы появятся в следующих обновлениях.».

9. **Прогресс чтения.** После листания нескольких страниц: `SELECT * FROM reading_progress WHERE user_id = <uid>` → одна запись с актуальными chapter_id / page_no. Throttle: быстрые 2 клика ▶ не должны создать второй UPDATE в пределах 5 с.

10. **«Моя полка».** `/shelf` → «🕒 Недавно» содержит этот фик с «гл.X стр.Y». Табы «📑 Закладки» и «❤️ Лайки» показывают фики, которые отметил.

11. **Дочитывание.** На последней странице последней главы появляется «✓ Дочитано». Нажать → alert «✓ Дочитано!», `SELECT reads_completed_count FROM fanfics WHERE id = X` вырос на 1; `SELECT * FROM reads_completed WHERE user_id = <uid> AND chapter_id = <last>` — запись появилась. Повторное нажатие counter не увеличит (идемпотентность).

12. **Правка approved-фика.** Автор правит текст главы → submit. Админ approve. Ожидание: в течение ≤ 60 с `ix_chapter_pages_chapter_page` показывает новые страницы (старые удалены воркером `repaginate_chapter`, который вызвал outbox-диспетчер по событию `fanfic.approved`). Открытие фика у читателя отдаёт обновлённый текст. Проверить:
    ```sql
    SELECT published_at, event_type, payload FROM outbox ORDER BY id DESC LIMIT 3;
    ```
    — `fanfic.approved` с `chapter_ids=[...]`, `published_at IS NOT NULL` после тика.

13. **Каталог → «🔥 Топ».** После нескольких ❤️ фик поднимается в ленте «Топ» (сортировка по `likes_count DESC`).

14. **Фильтр по фэндому.** `/catalog → 🏷 По фэндому → <выбрать>` → отображается лента только выбранного фэндома.

15. **Оглавление.** В режиме чтения → «📖 Оглавление» → клавиатура с главами. Клик на главу → страница 1 этой главы.

## Наблюдение

- `make logs bot worker` — метрики `reader_*`, `pagination_*` (в будущем Этапе 7 на Prometheus).
- При ошибках `edit_message_text` в логе `reader_edit_failed` — без алертов пользователю.

## Известные ограничения

- Полноценный outbox-диспетчер с ретраями / DLQ — минимальная реализация; расширим в Этапе 5–6.
- Поиск и фасетирование — Этап 4 (Meilisearch).
- Жалобы (⚠️) — заглушка до Этапа 5.
- Нет авто-отметки «дочитано за 30с просмотра» — только явная кнопка.
